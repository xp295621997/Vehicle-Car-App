# 小米汽车语音助手开发总结

## 一、项目概述

这是一个基于车联网的智能语音助手应用，支持：
- 语音对话交互
- 车辆控制与状态查询
- 图片地址识别并发送到车机
- 车辆配置知识库问答
- 场景化智能提醒

## 三、核心模块

### 1. Agent调度系统 (`dispatcher.ts`)

- **意图分类**：基于关键词匹配，支持多Agent分发
- **优先级机制**：车辆控制 > 车辆状态 > 知识库 > 场景引擎 > ...
- **上下文传递**：附件、历史记录、previousData

### 2. 手车互联 (`carPhoneInterop.ts`)

- **核心流程**：用户上传图片 → 图片理解 → 提取地址 → 发送到车机
- **多地址处理**：识别多个地址时让用户选择
- **多轮对话**：5轮上下文记忆，选择后继续对话

### 3. 场景引擎 (`scenarioEngine.ts` + `scenarioScheduler.ts`)

- **模拟场景**：按时间生成用户出行场景（早高峰、午休、夜间充电等）
- **定时任务**：每10分钟运行 + 每天10点推送
- **推理能力**：根据场景+车辆状态判断是否提醒

### 4. 车辆知识库RAG (`vehicleKnowledge.ts`)

- **本地知识库**：小米SU7 Ultra配置信息
- **关键词检索**：多维度匹配
- **相关度评分**：返回最佳答案

## 四、开发过程中的坑

### 1. CORS跨域问题

**问题**：浏览器直接调用第三方API失败
```
Failed to fetch
```

**原因**：浏览器安全策略限制

**解决方案**：
- 在Vite配置中添加代理中间件
- 前端调用 `/api/send-address-to-car` → 后端转发到真实API

```typescript
// vite.config.ts
server.middlewares.use('/api/send-address-to-car', async (req, res) => {
  // 转发请求到目标服务器
  const response = await fetch(targetUrl, { ... });
  const result = await response.json();
  res.end(JSON.stringify(result));
});
```

### 2. 意图识别优先级问题

**问题**：用户说"把这个图片里的地址发到车上去"，走到图片理解Agent

**原因**：
1. 附件在context中未正确传递到dispatcher
2. 意图分类优先级问题

**解决方案**：
- 修改VoiceChatPage，传递attachments到dispatch
- 在dispatcher中优先检查上下文状态（awaitingSelection）

### 3. 多轮对话上下文丢失

**问题**：用户选择地址后，再回复其他内容仍然在手车互联Agent中

**原因**：previousData获取逻辑错误

**解决方案**：
- 获取**最后一条assistant消息**的data，而不是最后一条消息
- 发送成功后返回 `type: 'send_success'` 标记
- dispatcher检测到该类型后重新分发

```typescript
// 获取之前的上下文数据（找最后一条assistant消息的data）
let previousData: any = undefined;
for (let i = conversationHistory.length - 1; i >= 0; i--) {
  if (conversationHistory[i].role === 'assistant' && conversationHistory[i].data) {
    previousData = conversationHistory[i].data;
    break;
  }
}
```

### 4. TypeScript接口导出问题

**问题**：
```
The requested module '/src/services/agents/scenarioEngine.ts' does not provide an export named 'Scenario'
```

**原因**：`interface`需要用`export type`导出

**解决方案**：
```typescript
// 错误
export { Scenario } from './scenarioEngine';

// 正确
export type { Scenario } from './scenarioEngine';
```

### 5. 独立服务与主应用冲突

**问题**：场景引擎在React组件中启动导致页面崩溃

**解决方案**：
- 将场景引擎拆分为独立Node.js服务（端口3001）
- 主应用通过HTTP调用获取场景

## 七、经验总结

1. **前后端分离的API调用**：涉及第三方API时一定要用后端代理，避免CORS问题
2. **上下文传递**：多轮对话需要显式传递history和previousData
3. **状态标记**：完成某个操作后用特定type标记，让dispatcher重新分发
4. **接口导出**：TypeScript中interface用`export type`导出
5. **独立服务**：定时任务/后台服务不应绑在React组件生命周期中
