## 四、核心功能实现

### 4.1 Agent 调度系统

这是整个应用的核心，负责理解用户意图并分发到正确的Agent。

#### 第一步：定义类型

创建 `src/services/agents/types.ts`：

```typescript
// Agent类型枚举
export enum AgentType {
  VEHICLE_CONTROL = 'vehicle_control',
  VEHICLE_STATUS = 'vehicle_status',
  CAR_PHONE_INTEROP = 'car_phone_interop',
  // ... 其他类型
}

// 意图类型枚举
export enum IntentType {
  VEHICLE_CONTROL = 'vehicle_control',
  VEHICLE_STATUS = 'vehicle_status',
  CAR_PHONE_INTEROP = 'car_phone_interop',
  // ... 其他意图
}

// Agent请求
export interface AgentRequest {
  query: string;
  context?: Record<string, any>;
  attachments?: Attachment[];
  onProgress?: (status: string) => void;
}

// Agent响应
export interface AgentResponse {
  content: string;
  action: string;
  agent: AgentType;
  intent: IntentType;
  success: boolean;
  error?: string;
}
```

#### 第二步：创建分发器

创建 `src/services/agents/dispatcher.ts`：

```typescript
import { IntentType, AgentRequest, AgentResponse, AgentType } from './types';

// 意图关键词映射
const INTENT_KEYWORDS: Record<IntentType, string[]> = {
  [IntentType.VEHICLE_CONTROL]: ['开车窗', '关车窗', '开空调', '锁车', ...],
  [IntentType.VEHICLE_STATUS]: ['续航', '电量', '胎压', ...],
  [IntentType.CAR_PHONE_INTEROP]: ['发送到车', '导航到', ...],
  // ...
};

// 意图分类
export function classifyIntent(query: string): IntentType {
  const lowerQuery = query.toLowerCase();

  for (const intent of Object.keys(INTENT_KEYWORDS)) {
    const keywords = INTENT_KEYWORDS[intent];
    if (keywords.some(kw => lowerQuery.includes(kw))) {
      return intent as IntentType;
    }
  }
  return IntentType.UNKNOWN;
}

// 分发器
export async function dispatch(request: AgentRequest): Promise<AgentResponse> {
  const { query, context, attachments } = request;

  // 1. 意图分类
  const intent = classifyIntent(query);

  // 2. 分发到对应Agent
  switch (intent) {
    case IntentType.VEHICLE_CONTROL:
      return await processVehicleControl(request);
    case IntentType.VEHICLE_STATUS:
      return await processVehicleStatus(request);
    case IntentType.CAR_PHONE_INTEROP:
      return await processCarPhoneInterop(request);
    // ...
    default:
      return { content: '抱歉，我无法理解', ... };
  }
}
```

#### 第三步：创建具体Agent

#### 第二步：创建代理（重要！）

在 `vite.config.ts` 中添加：

```typescript
export default defineConfig({
  plugins: [react(), tailwindcss()],
  server: {
    middlewares: [
      // 代理到真实API，绕过CORS
      async (req, res, next) => {
        if (req.url === '/api/send-address-to-car') {
          // 转发请求
          const result = await fetch('真实API地址', {
            method: 'POST',
            headers: {
              'Content-Type': 'application/json',
              'cookie': 'serviceToken=xxx', // 真实token
            },
            body: req.body,
          });
          res.end(JSON.stringify(await result.json()));
          return;
        }
        next();
      },
    ],
  },
});
```

```typescript
import { sendAddressToCar } from '../carPhoneApi';

export async function processCarPhoneInterop(request: AgentRequest): Promise<AgentResponse> {
  const { query, attachments } = request;

  // 1. 有图片附件 + 提到"地址" → 提取图片中地址
  if (attachments?.some(a => a.type === 'image') && query.includes('地址')) {
    // 调用图片理解API
    const imageDesc = await callImageUnderstandAPI(attachments[0].data);

    // 提取地址
    const addresses = extractAddresses(imageDesc);

    if (addresses.length === 1) {
      // 直接发送
      await sendAddressToCar(addresses[0]);
      return { content: '已发送到车机', ... };
    } else {
      // 多个地址，让用户选择
      return {
        content: `找到${addresses.length}个地址，请选择：\n${addresses.map((a, i) => `${i+1}. ${a}`).join('\n')}`,
        data: { awaitingSelection: true, addresses },
      };
    }
  }

  // 2. 用户选择地址
  if (query.match(/^\d+$/)) {
    // 解析选择，发送
  }

  return { content: '无法处理', ... };
}
```

### 4.3 多轮对话记忆

在 `VoiceChatPage.tsx` 中实现：

```typescript
// 状态：对话历史
const [conversationHistory, setConversationHistory] = useState<{role: 'user'|'assistant', content: string, data?: any}[]>([]);

// 发送消息时
const handleSend = async () => {
  // 获取上一条assistant的data（用于上下文）
  const previousData = conversationHistory
    .filter(h => h.role === 'assistant')
    .pop()?.data;

  // 调用dispatch，传递上下文
  const response = await dispatch({
    query: userInput,
    context: { previousData }
  });

  // 更新历史（保留最近5轮=10条）
  setConversationHistory(prev => [
    ...prev,
    { role: 'user', content: userInput },
    { role: 'assistant', content: response.content, data: response.data },
  ].slice(-10));
};
```

## 五、避坑指南（必读）

### ⚠️ 坑1：CORS跨域

**问题**：
```
Failed to fetch
```

**解决**：用后端代理，如上4.2节所示

---

### ⚠️ 坑2：上下文丢失

**问题**：多轮对话中previousData为undefined

**解决**：获取最后一条**assistant**消息的data
```typescript
// 错误
const previousData = conversationHistory[conversationHistory.length - 1]?.data;

// 正确
let previousData;
for (let i = conversationHistory.length - 1; i >= 0; i--) {
  if (conversationHistory[i].role === 'assistant') {
    previousData = conversationHistory[i].data;
    break;
  }
}
```

---

### ⚠️ 坑3：TypeScript接口导出

**问题**：
```
Module does not provide export named 'xxx'
```

**解决**：interface用`export type`导出
```typescript
// 错误
export { Scenario } from './types';

// 正确
export type { Scenario } from './types';
```

---

### ⚠️ 坑4：意图分类优先级

**问题**：用户说"发地址到车"，却走到图片理解

**解决**：在意图分类前检查上下文状态
```typescript
// 在dispatch函数开头
if (previousData?.awaitingSelection) {
  return processCarPhoneInterop(request); // 强制走这个
}
```

---

### ⚠️ 坑5：React组件中启动定时任务

**问题**：页面卡死

**解决**：拆分为独立Node.js服务（端口3001），不绑在React生命周期中

## 七、常见问题

### Q: 页面空白或白屏
A:
1. 检查控制台错误
2. 运行 `pnpm build` 检查编译
3. 重启 dev server: `lsof -i :5173 | grep node | awk '{print $2}' | xargs kill`

### Q: 接口返回401
A: token过期，需要从新抓包获取新的cookie

### Q: 图片理解失败
A: 检查MCP服务是否正常运行
