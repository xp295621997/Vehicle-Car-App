# 每日开发总结

## 3月23日

### 一、功能更新

**1. 修复每日总结生成功能**
- cron 定时任务修复：使用绝对路径 `/usr/local/bin/node` 解决 `node: command not found` 问题
- 更新 `auto-summary.cjs` 脚本：优先从 `conversations/日期.json` 读取数据（更可靠），回退到 `realtime.log` 解析
- 重新生成了 3月19日 和 3月23日 的每日总结

**2. 集成 MiniMax 图片生成 API**
- 后端：在 `vite.config.ts` 添加 `/api/image-generation` 端点
- 调用 `https://api.minimaxi.com/v1/image_generation`
- 前端：在 VoiceChatPage 添加 AI 图片生成弹窗
- 支持 6 种宽高比：`1:1`, `16:9`, `4:3`, `3:2`, `9:16`, `21:9`
- 生成图片在对话中显示，支持点击放大、右键保存

**3. 集成 MiniMax 语音合成 API (TTS)**
- 后端：在 `vite.config.ts` 添加 `/api/tts` 端点
- 调用 `https://api.minimaxi.com/v1/t2a_v2`
- 模型：`speech-2.8-hd`，声音：`male-qn-qingse`
- 前端：助手消息添加"朗读"按钮，点击生成并播放语音

### 二、问题修复

**1. 图片生成后消失问题**
- 原因：`messages.length` 在闭包中是旧值 + `setTimeout` 清除状态
- 修复：使用函数式更新 `setMessages(prev => [...prev, newMessage])`

**2. 朗读按钮交互问题**
- 所有按钮共享 loading 状态，导致一个按钮 loading 时全部按钮都显示 loading
- 修复：使用 `playingMessageId` 和 `loadingMessageId` 跟踪每条消息的独立状态

**3. 生成中状态显示优化**
- 添加按钮 pulse 动画效果
- 朗读按钮仅在 `loadingMessageId` 时 disabled，播放时可点击暂停

### 三、技能学习

**1. MiniMax API 学习**
- 图片生成 API (Text-to-Image)：`/v1/image_generation`
- 语音合成 API (Text-to-Audio)：`/v1/t2a_v2`
- 支持参数：model, prompt, aspect_ratio, voice_id, speed, emotion 等
- 特殊标记：停顿控制 `{#x#}`，语气词标签 `(laughs)`, `(sighs)` 等

**2. Figma 插件调研**
- 了解多个 Figma 代码导出插件：CodeSandbox、Figma to Code、Locofy、Anima
- 用于后续将设计文档转换为 React 代码

### 四、API/接口

**2. 语音合成 `/api/tts`**
