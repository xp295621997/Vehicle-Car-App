# 车辆控制指令接入流程

## 1. 指令格式规范

**接口地址：**
```
http://preview.tsp-mock.sc.iccc.mioffice.cn/test/dds-noauth
```

**请求方式：** POST (JSON)

**请求参数示例（打开前备箱）：**
```json
{
  "vid": "HXMQMEVC8B5MEBDB8",
  "cmdId": 33,
  "domain": "Hood",
  "device": "",
  "param": "open",
  "value": "2",
  "userId": "2421894891"
}
```

**关键参数说明：**
- `vid`: 车辆ID（需要从用户获取有效值）
- `cmdId`: 命令ID（不同功能不同）
- `domain`: 控制域（如 Hood、Trunk、Door 等）
- `param`: 参数（如 open、close、unlock 等）
- `value`: 参数值
- `userId`: 用户ID

**响应格式：**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "respCode": "00000",  // 00000 表示成功
    "respMsg": "成功",
    "respData": { ... }
  }
}
```

## 2. 服务端配置

### 2.1 创建 API 服务模块

在 `src/services/carApi.ts` 中定义：

```typescript
// 车辆配置
const CAR_CONFIG = {
  vid: 'HXMQMEVC8B5MEBDB8',
  userId: '2421894891',
};

// 发送车辆控制命令
export async function sendCarCommand(cmdId: number, domain: string, param: string, value: string) {
  const command = {
    vid: CAR_CONFIG.vid,
    cmdId,
    domain,
    device: '',
    param,
    value,
    userId: CAR_CONFIG.userId,
  };

  const response = await fetch('/api/car-control', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(command),
  });

  const result = await response.json();

  // 检查业务层面的响应码
  if (result.data && result.data.respCode && result.data.respCode !== '00000') {
    throw new Error(`API错误: ${result.data.respMsg || result.data.respCode}`);
  }

  return result;
}

// 预定义命令
export const carActions = {
  openHood: () => sendCarCommand(33, 'Hood', 'open', '2'),
  // ... 其他命令
};
```

### 2.2 配置 Vite 服务端转发（解决 CORS）

在 `vite.config.ts` 中添加服务端转发端点：

```typescript
// 车辆控制 API 端点 - 服务端转发避免 CORS
server.middlewares.use('/api/car-control', async (req, res) => {
  if (req.method === 'POST') {
    let body = '';
    req.on('data', chunk => body += chunk);
    req.on('end', async () => {
      try {
        const carData = JSON.parse(body);
        console.log('[Car API] 收到请求:', JSON.stringify(carData));

        // 直接转发到目标服务器
        const targetUrl = 'http://preview.tsp-mock.sc.iccc.mioffice.cn/test/dds-noauth';
        const response = await fetch(targetUrl, {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify(carData),
        });

        const result = await response.json();
        console.log('[Car API] 目标服务器返回:', JSON.stringify(result));

        res.setHeader('Content-Type', 'application/json');
        res.end(JSON.stringify(result));
      } catch (error: any) {
        console.error('[Car API] 请求失败:', error);
        res.status(500).json({ error: error.message });
      }
    });
  } else {
    res.status(405).json({ error: 'Method not allowed' });
  }
});
```

### 4.3 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| HTTP 403 / CORS 错误 | 跨域限制 | 使用服务端转发 `/api/car-control` |
| `respCode: A0411` | vid 无效 | 确认有效的车辆ID |
| `respCode: 非00000` | 业务错误 | 查看 `respMsg` 具体信息 |

## 6. 注意事项

- 响应码是 `00000`（5个0）不是 `0000`（4个0）
- 需要通过服务端转发避免 CORS 问题
- 每次修改后需要重启开发服务器使配置生效
